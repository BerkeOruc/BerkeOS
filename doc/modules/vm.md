# BexVM Virtual Machine

## Overview

BexVM is BerkeOS's bytecode interpreter for executing Python programs compiled to .bex format. It provides a stack-based virtual machine with support for basic data types, control flow, and system calls.

## Architecture

### VM Components

- **Stack**: 256-element i64 stack for operands
- **Locals**: 32 local variables per function
- **Call Stack**: 64-level function call stack
- **Arrays**: 32 dynamic arrays (max 64 elements each)
- **Dictionaries**: 32 hash tables (max 32 entries each)
- **Exception Handling**: 16-level exception stack

### Data Types

- **Integer**: 64-bit signed integers
- **Float**: 64-bit IEEE 754 (stored as integer)
- **String**: UTF-8 byte arrays
- **Boolean**: Single byte (0/1)
- **Array**: Dynamic arrays of integers
- **Dictionary**: Key-value mappings

### Bytecode Format

```
.bex File Structure:
- Magic: 0x42455831 ("BEX1")
- Version: u16 (currently 1)
- Name Length: u16
- Program Name: [u8; name_len]
- Constants Table: Variable-length
- Functions Table: Variable-length
- Bytecode: Variable-length instructions
```

### Instruction Set

#### Stack Operations

- **NOP (0)**: No operation
- **PUSH (1)**: Push constant onto stack
- **POP (2)**: Remove top of stack
- **DUP (3)**: Duplicate top of stack
- **SWAP (4)**: Swap top two elements

#### Arithmetic

- **ADD (10)**: Add top two elements
- **SUB (11)**: Subtract top two elements
- **MUL (12)**: Multiply top two elements
- **DIV (13)**: Divide top two elements
- **MOD (14)**: Modulo operation

#### Control Flow

- **JMP (20)**: Unconditional jump
- **JZ (21)**: Jump if zero
- **JNZ (22)**: Jump if not zero
- **CALL (23)**: Function call
- **RET (24)**: Return from function

#### Data Operations

- **LOAD (30)**: Load local variable
- **STORE (31)**: Store to local variable
- **ARRAY_NEW (32)**: Create new array
- **ARRAY_GET (33)**: Get array element
- **ARRAY_SET (34)**: Set array element
- **DICT_NEW (35)**: Create new dictionary
- **DICT_GET (36)**: Get dictionary value
- **DICT_SET (37)**: Set dictionary value

#### I/O Operations

- **PRINT (40)**: Print value to console
- **SYSCALL (41)**: Invoke system call

### Function Calls

- **Call Stack**: Saves return address and local variables
- **Parameter Passing**: Via stack
- **Recursion**: Supported up to 64 levels

### Exception Handling

- **Try/Catch**: Exception frames on stack
- **Throw**: Set current exception
- **Unwind**: Stack unwinding on exception

## System Integration

### Syscall Interface

BexVM programs access OS services through syscalls:

```rust
// Example: Print to console
syscall(SYS_WRITE, 1, buffer_ptr, buffer_len)
```

### File I/O

- **File Operations**: fopen, fread, fwrite, fclose
- **Directory Operations**: mkdir, delete, exists
- **Path Resolution**: Relative to current drive

### Graphics Support

- **Framebuffer**: Direct pixel access
- **Text Rendering**: Built-in font support
- **Color Operations**: RGB color values

## Limitations

- **Memory**: Fixed-size stacks and heaps
- **Types**: No user-defined types
- **Performance**: Interpreted execution
- **Concurrency**: Single-threaded
- **I/O**: Blocking operations only

## Compilation

BexVM bytecode is generated by the BerkeBex compiler from Python source:

```
Python Source → AST → IR → Bytecode → .bex File
```

The compiler handles:
- **Syntax Analysis**: Python subset parsing
- **Type Inference**: Basic type checking
- **Optimization**: Constant folding, dead code elimination
- **Code Generation**: Stack machine bytecode
